# 星浪音乐网站建设记录

## 项目目标
- 把现有站点从通用商务模板，逐步整理成真正像“星浪音乐”的音乐平台网站。
- 优先处理首页视觉、品牌统一、内容完整度、中文化和音乐属性表达。
- 每次修改前先看页面，修改后再回看效果，避免只改源码不看界面。

## 固定工作方法
1. 先打开页面，看当前真实界面和视觉层次。
2. 找出品牌错位、文案混乱、模块不协调、样式老旧等问题。
3. 按页面优先级推进：首页 > 导航/品牌 > 核心功能页 > 细节页。
4. 每次只推进一批明确改动，改完后重新预览。
5. 把新增、删除、替换、保留的内容写进这个记录。
6. 每小时汇报必须对应到施工清单和施工日志，不能报空话。

## 经验积累
### 经验 1：先看页面，再改源码
- 只看源码容易误判真实视觉效果。
- 前端改版必须遵循：看界面 -> 修改 -> 再看界面 -> 继续优化。

### 经验 2：品牌统一优先级很高
- logo、站名、导航文案、首屏标题不统一时，页面再花哨也会像拼接模板。
- 先统一品牌，再谈内容丰富和样式精修。

### 经验 3：音乐站要先有“音乐感”
- 首屏标题、模块命名、图片、按钮文案都要服务于“音乐上传、创作、发现、交流”这几个核心关键词。
- 如果首页第一眼像企业咨询站，就说明方向偏了。
