## 规则（执行要求）

- 用户每次新交代一个任务，都必须立刻写进本文件的“当前未完成任务（实时队列）”。
- 任务完成后，要立即在本文件中转入或标注到“已完成”，不能继续挂在未完成队列里。
- 用户验收后如果指出哪里不行，该反馈视为一个新任务，要重新记入未完成队列，再继续修改。
- 做手机端页面时，完成后要用截图和站内其他页面对比，确保字体和整体视觉尺寸一致。
- 如果汇报后用户没有回复“停止/结束”，默认自动继续往下做，直到当前工程全部完成。
- 当前明确任务做完后，如用户仍未叫停，则继续主动检查网站各页面，发现不完善处就作为新任务记入并继续优化，直到用户明确说“不用完善了/工程结束”。
- 每完成一项任务，也必须同步写入当日 `memory/YYYY-MM-DD.md`，防止切换模型后重复工作。
